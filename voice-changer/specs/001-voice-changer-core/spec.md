# Feature Specification: Personal Voice Changer

**Feature Directory**: `specs/001-voice-changer-core`

**Created**: 2026-06-26

**Status**: Clarified

## User Scenarios & Testing

<!--
  User stories are ordered by priority. Each story is independently testable
  and delivers standalone value.

### User Story 1 — Voice Preview in Test Mode (Priority: P1)

男性ユーザーが通話前に自分の加工後の音声を確認し、女声として自然に聞こえるよう調整できる。

**Why this priority**: これがなければ通話で使う前に音質を確認できない。全機能の前提となるMVP。

**Independent Test**: アプリを起動してテストモードを選択し、マイクに話しかけるだけで加工後の声がヘッドフォンから聞こえれば成立。通話アプリもVB-Cableも不要。

**Acceptance Scenarios**:

1. **Given** アプリが起動しマイクが接続されている、**When** テストモードを選択してマイクに話しかける、**Then** 加工後の女声がヘッドフォンからリアルタイムで聞こえる
2. **Given** テストモードが動作中、**When** ピッチスライダーを動かす、**Then** 即座に声の高さが変化する
3. **Given** テストモードが動作中、**When** 声質スライダーを動かす、**Then** ピッチとは独立して声の質（女性らしさ）が変化する
4. **Given** 処理品質モードを変更する、**When** モードAを選択する、**Then** 1秒以内に自然な女声が聞こえる

---

### User Story 2 — Female Voice in Discord Calls (Priority: P2)

男性ユーザーがDiscord等の通話アプリで、相手に女声として認識される音声を送れる。

**Why this priority**: アプリの最終目的。US1で調整した設定をそのまま通話で活かせる。

**Independent Test**: 通話モードに切り替えてDiscordのマイクとして設定したデバイスを選択し、Discordの相手が女声を聞けば成立。

**Acceptance Scenarios**:

1. **Given** テストモードで調整済みの設定がある、**When** 通話モードに切り替える、**Then** 出力先が仮想マイクデバイスに切り替わり、自分のヘッドフォンへのモニタリングが止まる
2. **Given** 通話モードで動作中、**When** Discordが仮想マイクからの音声を受信する、**Then** 相手には加工後の女声が届く
3. **Given** 通話モードで動作中、**When** UIを確認する、**Then** 「ボイスチェンジ稼働中」インジケーターが表示されており、変換が動作していることを視覚的に確認できる
4. **Given** 仮想マイクドライバー（VB-Cable）がインストールされていない、**When** アプリを起動する、**Then** セットアップ案内が表示され、インストール手順とDiscordでの設定方法が分かる

---

### User Story 3 — Effect & Preset Management (Priority: P3)

ユーザーがエフェクト設定をプリセットとして保存・切替でき、通話中でもリアルタイムに声の質を変更できる。

**Why this priority**: 快適な運用のために必要だが、US1・US2が動作していれば最低限の利用は可能。

**Independent Test**: プリセットを保存して別のプリセットに切り替えた後、元のプリセットを読み込んで同じ設定が復元されれば成立。

**Acceptance Scenarios**:

1. **Given** 気に入った設定が見つかった、**When** 「保存」ボタンを押してプリセット名を入力する、**Then** 設定がプリセットとして永続化される
2. **Given** 複数のプリセットが保存されている、**When** 通話中にプリセットを切り替える、**Then** 即座に声の質が変わる（中断なし）
3. **Given** ノイズゲートが有効、**When** 周囲の環境音が一定レベル以下になる、**Then** その音は出力されない
4. **Given** パススルーモードが無効、**When** パススルーボタンを押す、**Then** 即座にエフェクトなしの素の声に切り替わる
5. **Given** プリセットをエクスポートしたい、**When** エクスポートボタンを押す、**Then** 設定ファイルがダウンロードできる

---

### User Story 4 — Easy Launch & Setup (Priority: P4)

ユーザーがファイルをダブルクリックするだけでアプリが起動し、ブラウザでコントロール画面が開く。

**Why this priority**: 利便性。US1〜3が動作した後の仕上げ。

**Independent Test**: 起動ファイルをダブルクリックしてブラウザが自動で開き、コントロール画面が表示されれば成立。

**Acceptance Scenarios**:

1. **Given** アプリが未起動、**When** 起動ファイルをダブルクリックする、**Then** 音声エンジンが起動しブラウザでコントロール画面が自動で開く
2. **Given** アプリが初めて起動された（VB-Cable未インストール）、**When** コントロール画面を開く、**Then** VB-Cableのセットアップ案内が表示される
3. **Given** アプリが起動した、**When** コントロール画面を確認する、**Then** デフォルトのプリセット（ピッチ+10・声質1.4倍）が選択済みになっている

---

### Edge Cases

- マイクデバイスが接続されていない状態でアプリを起動した場合、適切なエラーメッセージが表示される
- 通話中にマイクデバイスが物理的に切断された場合、アプリがクラッシュせず安全に停止する
- プリセットファイルが破損していた場合、エラーを表示してデフォルト設定で起動する
- 処理品質を自然さ重視（モードA）に設定した場合でも1秒以内に音声が届く
- 処理品質をバランス（モードC）に設定した場合は200ms以内に音声が届く
- 処理品質を低遅延（モードB）に設定した場合は50ms以内に音声が届く

## Requirements

### Functional Requirements

- **FR-001**: ユーザーは音声のピッチ（高さ）を±24半音の範囲で変更できる。デフォルト: +10半音
- **FR-002**: ユーザーは声質（声道長）を×0.5〜×2.0の範囲で変更できる。デフォルト: ×1.4
- **FR-003**: アプリはテストモード（自分のヘッドフォンへ出力）と通話モード（仮想マイクデバイスへ出力）を切り替えられる
- **FR-004**: 通話モードでは自分のヘッドフォンへのモニタリングを行わない。UIのインジケーターで稼働確認できる
- **FR-005**: アプリは3段階の処理品質モードを持つ: 自然さ重視（≤1000ms）・低遅延（≤50ms）・バランス（≤200ms）
- **FR-006**: ノイズゲート・リバーブ・ロボットボイスの3種類のエフェクトを個別にON/OFF・強度調整できる
- **FR-007**: 利用可能なマイク・出力デバイスの一覧を表示し、個別に選択できる
- **FR-008**: 仮想マイクドライバー（VB-Cable）が未検出の場合、インストール案内をUI上に表示する
- **FR-009**: 設定をプリセットとして保存・読み込み・削除できる。プリセットはアプリフォルダに永続化される
- **FR-010**: プリセットをファイルとしてエクスポート・インポートできる（バックアップ・移行用）
- **FR-011**: パススルーモードで全エフェクトを即座にバイパスし、素の声を出力できる
- **FR-012**: 起動ファイルをダブルクリックするだけで音声エンジンが起動し、ブラウザでUI画面が自動的に開く
- **FR-013**: 音量レベルと稼働状態をリアルタイムで画面上に表示する

### Key Entities

- **Preset**: ピッチ・声質・処理品質モード・エフェクト設定のスナップショット。名前付きで保存され切替可能
- **QualityMode**: 処理の精度と遅延のトレードオフを定義するモード（自然さ重視 / 低遅延 / バランス）
- **OutputMode**: 音声の出力先と自己モニタリングの有無を定義するモード（テスト / 通話）
- **AudioDevice**: マイクまたはスピーカー/仮想デバイス。入力・出力を個別に選択できる

## Success Criteria

### Measurable Outcomes

- **SC-001**: 自然さ重視モードで話した声が1秒以内に加工されて出力される
- **SC-002**: 低遅延モードで話した声が50ms以内に加工されて出力される
- **SC-003**: バランスモードで話した声が200ms以内に加工されて出力される
- **SC-004**: デフォルト設定（ピッチ+10・声質×1.4）で男声を入力したとき、聞いた人が女声と認識できる
- **SC-005**: コントロール画面でパラメータを変更してから100ms以内に設定が反映される（音声への反映は各モードの遅延上限に従う）
- **SC-006**: 30分間の連続動作でクラッシュが発生しない
- **SC-007**: VB-Cableが未インストールの状態でも、テストモードでの声確認機能が使用できる

---

## Assumptions

- ユーザーはWindows 10/11を使用している（実行環境）
- 開発はUbuntu/WSL2環境で行い、音声デバイスを使わないテストはUbuntu上で実施する
- Discordなどの通話アプリは仮想マイクデバイスを入力として使用できる
- ユーザーはVB-Cableを事前に知らない可能性があるため、アプリが案内を提供する
- コントロール画面はローカル環境（localhost）のブラウザからのみアクセスする（外部公開しない）
- 処理品質モードAが通常の環境で1秒以内に動作することが前提。達成困難な環境ではモードBを使用する
- 複数人での同時利用はスコープ外（個人利用のみ）
- macOS / Linux 上での実行はスコープ外
