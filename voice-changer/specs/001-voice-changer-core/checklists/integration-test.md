## Phase 1 — テストモードで女声確認 (US1 / SC-001〜003 / SC-004)

> T071 ステップ 3 / FR-001〜003 / FR-005 / FR-011 / FR-013

### 基本動作

- [ ] P1-1: 出力モード「**Test（Headphones）**」を選択する
- [ ] P1-2: 品質モード **B（低遅延 ≤50ms）** を選択してマイクに話す → ヘッドフォンから女声が聞こえる **（SC-002）**
- [ ] P1-3: 品質モード **C（バランス ≤200ms）** に切替 → 聞こえ方が変わる、遅延が知覚できる範囲か確認 **（SC-003）**
- [ ] P1-4: 品質モード **A（自然さ重視 ≤1000ms）** に切替 → 1秒以内に女声が聞こえる **（SC-001）**
- [ ] P1-5: デフォルト設定（ピッチ+10・声質×1.4）で聞いた人が女声と認識できる **（SC-004）**

### パラメータ調整（US1 AC-2〜3）

- [ ] P1-6: ピッチスライダーを動かす → 即座に声の高さが変わる **（US1 AC-2）**
- [ ] P1-7: 声質スライダーを動かす → ピッチとは独立して声の質が変わる **（US1 AC-3）**

### パススルー（FR-011 / US3 AC-4）

- [ ] P1-8: 「**Passthrough**」ボタンを押す → 即座にエフェクトなしの素の声に切り替わる **（US3 AC-4）**
- [ ] P1-9: 再度「Passthrough」を押す → 女声処理が再開する

### リアルタイム表示（FR-013）

- [ ] P1-10: 話すと音量レベルメーターが反応する
- [ ] P1-11: 「稼働中」インジケーターが緑のパルスで点灯している

## Phase 3 — プリセット管理 (US3 / FR-009〜010)

> T071 ステップ 5 / FR-009〜010

### 保存と読み込み（US3 AC-1〜2）

- [ ] P3-1: 品質モード C・ピッチ+8 に設定する
- [ ] P3-2: プリセット名「my_test」を入力して「**保存**」ボタンを押す → プリセット一覧に追加される **（US3 AC-1）**
- [ ] P3-3: 「**default**」プリセットに切り替える → 設定がデフォルトに変わる
- [ ] P3-4: 「my_test」プリセットに切り替える → ピッチ+8・モードCに復元される **（US3 AC-1）**
- [ ] P3-5: 通話中（モード動作中）にプリセットを切り替える → 中断なく即座に声が変わる **（US3 AC-2）**

### エクスポート / インポート（US3 AC-5 / FR-010）

- [ ] P3-6: 「**Export**」ボタンを押す → YAML ファイルがダウンロードされる **（US3 AC-5）**
- [ ] P3-7: ダウンロードした YAML ファイルを「**Import**」する → プリセット一覧に追加される

### エフェクト（FR-006 / US3 AC-3）

- [ ] P3-8: 「**Noise Gate**」を ON にする → 小さな環境音が出力されなくなる **（US3 AC-3）**

- [ ] P3-10: 「**Robot Voice**」を ON にする → ロボット風の声になる

## Phase 5 — エッジケース確認

> spec.md Edge Cases / SC-006〜007

### 安定性（SC-006）

- [ ] P5-1: テストモードまたは通話モードで **30分間** 連続動作させてクラッシュしない **（SC-006）**

### VB-Cable なしでもテストモード使用可能（SC-007）

- [ ] P5-2: VB-Cable 未接続でも「Test モード」でヘッドフォンから女声が聞こえる **（SC-007）**

### マイク切断（Edge Case）

- [ ] P5-3: 通話中にマイクを物理的に抜く → アプリがクラッシュせず安全に停止する（エラーメッセージ表示）

### 破損プリセット（Edge Case）

- [ ] P5-4: プリセット YAML を手動で壊してアプリを再起動する → エラー表示してデフォルト設定で起動する
